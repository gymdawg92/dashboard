Privacy Policy

William's Command Center · Effective 2026-04-26

1. Who this is

William's Command Center (the "App") is a private, single-user dashboard built and operated by Rooted Financial for internal use by William Hatcher. The App connects to QuickBooks Online via Intuit's official OAuth API and displays financial summaries on a personal dashboard.

The App is not offered to the public, has no user registration, and is not intended for use by anyone other than the operator. If you reached this page expecting a consumer service, this isn't one — you can close the tab.

2. What data we access

The App reads the following from the operator's QuickBooks Online company, with prior authorization granted via Intuit OAuth:

  • Profit & Loss reports (year-to-date and monthly breakdowns)
  • Balance Sheet reports (specifically cash on hand)
  • Accounts Receivable Aging Summary (outstanding receivables)

The App does not create, modify, or delete any data in QuickBooks. It does not access customer PII, transaction-level detail, employee records, or any data outside the reports listed above.

3. Where data is stored

The App uses Supabase (a hosted Postgres + Edge Function platform operated by Supabase, Inc.) for storage and backend logic. Specifically:

  • OAuth tokens (refresh + access tokens needed to call Intuit on the operator's behalf) are stored in a private database table accessible only via service-role credentials.
  • Financial snapshots (parsed report values) are stored in a private database table to enable fast page loads without re-fetching from Intuit each time.

Both tables have row-level security enabled and are not exposed to the public Supabase API. Data is encrypted in transit (TLS) and at rest using Supabase's standard infrastructure.

4. Who has access

Only the operator (William Hatcher) has access to the dashboard, the underlying Supabase project, and the Intuit Developer app. No third party — including Intuit and Supabase — is given access to the data beyond what is required to operate their services on the operator's behalf.

5. Retention

Financial snapshots are retained until the operator manually overwrites them (each refresh replaces the prior snapshot) or deletes the row. OAuth tokens are retained until the operator revokes the connection from Intuit's My Apps page or deletes the row from the database, at which point all stored credentials become unusable.

6. Sharing and third parties

The App does not sell, trade, or share data with anyone. The only third parties involved are:

  • Intuit — source of the financial data, accessed via OAuth using the credentials the operator approved.
  • Supabase, Inc. — infrastructure provider where the database and Edge Functions run.
  • GitHub Pages — hosts the static frontend HTML that runs in the operator's browser.

Each of these providers maintains their own privacy practices. We rely on their standard security and processing terms.

7. Cookies and tracking

The App stores a single value in localStorage in the operator's browser to remember whether the password gate has been unlocked, and a copy of recent dashboard data for offline display. The App does not use cookies, analytics, advertising trackers, or fingerprinting.

8. Revoking access

The operator can revoke the App's access to QuickBooks at any time via the Intuit "My Apps" page at https://appcenter.intuit.com/connect/myaccount/. Once revoked, stored refresh tokens can no longer be used to fetch data, and the operator can also delete the row from the credentials table to wipe local copies.

9. Changes

If this policy changes, the updated version will replace this page with a new "Effective" date. Because this is a single-user app, there is no notification list to email.

10. Contact

Questions about this policy or the App's data handling can be directed to info@rootedfinancial.co.
